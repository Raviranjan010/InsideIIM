# Core AWS Services: The Pillars of Infrastructure

Amazon Web Services (AWS) is the largest cloud provider. To discuss cloud systems in an interview, you must understand the purposes of its four core services.

---

## 1. Amazon EC2 (Elastic Compute Cloud)
*   **One-Sentence Purpose:** A virtual server in the cloud that you can configure and run like a standard computer.
*   **Realistic Scenario:** Running a custom background Node.js server script that needs to stay active 24/7 to scrape financial reports.

## 2. Amazon S3 (Simple Storage Service)
*   **One-Sentence Purpose:** A secure, massive folder in the cloud used to store static files like images, PDFs, and data backups.
*   **Realistic Scenario:** Storing the final PDF reports generated by our investment assistant so users can download them via a unique link.

## 3. AWS Lambda (Serverless Compute)
*   **One-Sentence Purpose:** A service that runs your code only when triggered by an event, billing only for the milliseconds of execution.
*   **Realistic Scenario:** Running a daily script at midnight to fetch financial news for the cache table, shutting down automatically when completed.

## 4. Amazon RDS (Relational Database Service)
*   **One-Sentence Purpose:** A managed database hosting service that runs database engines (like PostgreSQL) and handles backups automatically.
*   **Realistic Scenario:** Storing our user profiles, message history threads, and cached stock research inside a production-grade PostgreSQL database.
