---

## 🔑 Answer Key

1. **C** — The client is the device or application (like Chrome) requesting data.
2. **B** — DNS translates readable addresses like `wikipedia.org` to machine-readable IP addresses.
3. **B** — Files are chopped into packets, sent separately, and reassembled at the destination.
4. **D** — Headers hold metadata. The actual content is stored in the Body.
5. **C** — `POST` is used to send data to create resources (e.g. submitting a form).
6. **C** — `4xx` is for client errors (like `404 Not Found`). `5xx` codes are server errors.
7. **B** — `404 Not Found` indicates the resource doesn't exist on the server.
8. **C** — `ls` stands for "list" (lists files and folders).
9. **B** — The double dot `..` represents the parent folder.
10. **B** — You call the interpreter `node` followed by the filename `app.js`.
11. **C** — A repository is a directory monitored by Git for history changes.
12. **B** — Staging area lets you prepare specific file changes for the next commit.
13. **C** — `git init` initializes a blank Git repo.
14. **B** — `-m` stands for message, letting you label your save snapshot.
15. **D** — `git push` sends local commits up to a remote host.
